# ベルトスクロールアクション ゲーム仕様書

## 1. 概要

| 項目 | 内容 |
|------|------|
| ジャンル | ベルトスクロールアクション（Beat 'em Up） |
| エンジン | Godot 4.x (GDScript) |
| グラフィック | 2Dスプライト（ドット絵風） |
| 画面解像度 | 384×224（16:9 レトロ風）、ウィンドウは2〜3倍表示 |
| プレイヤー人数 | 1〜2人（ローカル協力プレイ） |
| 入力 | キーボード + ゲームパッド対応 |
| ターゲットFPS | 60fps |

## 2. ゲームフロー

```
タイトル画面
  ├─ 1P START → キャラクター選択 → ゲーム開始
  ├─ 2P START → キャラクター選択（×2）→ ゲーム開始
  └─ OPTIONS → 設定画面

ゲーム中
  ステージ1 → ステージ2 → ステージ3 → エンディング
  （各ステージ: 道中 → 中ボス(任意) → ボス）

ゲームオーバー → コンティニュー選択 → タイトルへ / 再開
```

## 3. プレイヤーキャラクター

### 3.1 キャラクター一覧（3体）

| キャラクター | タイプ | 特徴 |
|-------------|--------|------|
| ファイター | バランス型 | 攻撃力・速度・リーチが平均的 |
| グラップラー | パワー型 | 攻撃力・投げが強力、移動速度が遅い |
| スピードスター | スピード型 | 移動速度・コンボ回数が多い、攻撃力が低い |

### 3.2 基本パラメータ

| パラメータ | 説明 |
|-----------|------|
| HP | ヒットポイント（0で残機消費） |
| 攻撃力 | 基本攻撃ダメージ |
| 移動速度 | X・Y軸方向の移動速度 |
| 残機 | デフォルト3。0でゲームオーバー |

### 3.3 アクション一覧

| アクション | 入力 | 説明 |
|-----------|------|------|
| 移動 | 方向キー / 左スティック | 8方向移動（X軸: 横移動、Y軸: 奥行き移動） |
| 通常攻撃 | 攻撃ボタン | 連打で最大3〜4段コンボ |
| ジャンプ | ジャンプボタン | その場ジャンプ |
| ジャンプ攻撃 | 空中+攻撃ボタン | 空中から攻撃（着地時に隙） |
| ダッシュ | 方向キー2回入力 | 短距離ダッシュ移動 |
| ダッシュ攻撃 | ダッシュ中+攻撃ボタン | ダッシュ状態からの攻撃 |
| 掴み | 敵に密着+攻撃ボタン | 敵を掴む（掴み後に派生あり） |
| 掴み→打撃 | 掴み中+攻撃ボタン | 掴んだ敵に膝蹴り等 |
| 掴み→投げ | 掴み中+方向+攻撃ボタン | 方向指定で投げ飛ばす（他の敵にも当たる） |
| 必殺技 | 攻撃+ジャンプ同時押し | 周囲を攻撃（HP消費型、緊急回避用） |
| アイテム拾い | アイテム上で攻撃ボタン | 落ちているアイテムを拾う |
| 武器攻撃 | 武器所持中+攻撃ボタン | 拾った武器で攻撃（耐久度あり） |

## 4. 敵キャラクター

### 4.1 ザコ敵

| 敵名 | タイプ | 行動パターン |
|------|--------|-------------|
| チンピラ | 近接型 | 近づいて殴る。最も基本的な敵 |
| ナイフマン | 近接武器型 | ナイフで攻撃。リーチがやや長い |
| パンクス | 突進型 | ダッシュ突進してくる。ガード不可 |
| 火炎瓶兵 | 遠距離型 | 火炎瓶を投げてくる。地面に火が残る |
| 大男 | 重量型 | HPが高くのけぞりにくい。掴み不可 |

### 4.2 ボス敵

| ステージ | ボス名 | 特徴 |
|---------|--------|------|
| Stage 1 | バウンサー | 連続パンチ＋タックル。パターンが読みやすい入門ボス |
| Stage 2 | クノイチ | 高速移動＋分身＋手裏剣。回避重視の戦い |
| Stage 3 | ボスX（最終ボス） | 第1形態：格闘、第2形態：武器使用。多彩な攻撃パターン |

### 4.3 敵AIの基本仕様

- **同時攻撃制限**: 画面内で同時にプレイヤーを攻撃できる敵は最大2〜3体。残りは周囲で待機
- **包囲行動**: プレイヤーの左右・上下から挟み込むように移動
- **難易度調整**: ステージが進むほど、攻撃頻度・HP・同時出現数が増加

## 5. ステージ構成

### 5.1 共通仕様

- 横スクロール＋奥行き移動が可能なフィールド
- スクロールはプレイヤーの前方移動に追従
- 特定地点で敵が出現し、全滅するとスクロールロック解除
- 各ステージの最後にボス戦

### 5.2 ステージ一覧

| ステージ | テーマ | 特徴 |
|---------|--------|------|
| Stage 1: 街 | 夜の繁華街 | チュートリアル的。基本敵のみ。壊せるオブジェクト（ドラム缶等）からアイテム出現 |
| Stage 2: 倉庫 | 港の倉庫地帯 | 落とし穴（穴に落とすギミック）、コンテナで狭い通路。遠距離敵が出現 |
| Stage 3: アジト | 敵のアジト | エレベーター戦闘（閉鎖空間で連戦）。最終ボス戦 |

## 7. スコアシステム

| 行動 | スコア |
|------|--------|
| ザコ敵撃破 | 100〜500（敵種別） |
| ボス撃破 | 5000〜20000 |
| コンボボーナス | コンボ数 × 50 |
| ノーダメージボーナス | ステージクリア時にダメージ0なら10000 |
| タイムボーナス | 残りタイム × 10 |
| アイテムスコア | アイテム固有値 |

- ステージクリア時にリザルト画面表示
- 一定スコアで1UP

## 8. 2人協力プレイ

- ローカル同一画面での協力プレイ
- 1Pはキーボード or パッド1、2Pはパッド2
- 味方への攻撃は**なし**（フレンドリーファイアOFF）
- 掴み中の敵に味方が攻撃可能（協力コンボ）
- 2人プレイ時は敵の出現数が1.5倍に増加

### 10.2 コアシステム設計

#### キャラクター状態遷移（ステートマシン）

```
Idle ←→ Walk
 ↓↑       ↓
Attack → Combo1 → Combo2 → Combo3
 ↓
Jump → JumpAttack
 ↓
Dash → DashAttack
 ↕
Grab → GrabAttack / Throw
 ↓
Hit → Knockdown → GetUp
 ↓
Special (必殺技)
 ↓
Dead
```

#### 奥行き（Y軸）の処理

- ゲームフィールドは擬似的な奥行きを持つ（Y座標で奥行き表現）
- 攻撃の当たり判定はX座標に加え、Y座標の近さ（奥行き一致度）も考慮
- Y座標によるスプライトの描画順ソート（手前のキャラが前面に描画）
- 奥行き許容幅: ±8px（この範囲内なら攻撃がヒット）

#### 当たり判定システム

- **Hitbox**: 攻撃側が出す判定エリア（Area2D）
- **Hurtbox**: 被弾側の判定エリア（Area2D）
- 衝突レイヤーでプレイヤー攻撃 / 敵攻撃を分離
- ヒットストップ: 攻撃ヒット時に2〜4フレームの一時停止

#### スクロールカメラ

- プレイヤー（2人の場合は中間点）を追従
- スクロールロック区間（敵全滅まで進行不可）
- Y軸の可動範囲を制限してフィールドの奥行き表現
