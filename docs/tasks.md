- [ ] **P1-04** Autoload スケルトン作成 — `input_manager.gd`（デバイスマッピング＋入力取得メソッド）
- [ ] **P1-05** Autoload スケルトン作成 — `score_manager.gd`（スコア変数＋加算・リセット）

- [ ] **P1-07** Autoload 4つを `project.godot` に登録

## Phase 2: プレイヤー基本動作

- [ ] **P2-01** `character_data.gd` カスタムリソース定義（name, max_hp, attack_power, move_speed, dash_speed, jump_force, combo_count, grab_power, special_hp_cost, sprite_frames）
- [ ] **P2-02** 3キャラ分の `.tres` 作成（fighter.tres / grappler.tres / speedster.tres にパラメータ値を設定）
- [ ] **P2-03** プレイヤーシーン構築（`player.tscn`: CharacterBody2D + AnimatedSprite2D + Shadow + CollisionShape + Hitbox + Hurtbox + GrabPoint + StateMachine + AnimationPlayer）
- [ ] **P2-04** `player_controller.gd` 作成（ルートスクリプト。キャラデータ読み込み、HP管理、向き管理の基盤）
- [ ] **P2-05** ステートマシン基盤 — `player_state_machine.gd`（ステート登録・遷移・current_state 管理）
- [ ] **P2-06** ステート基底クラス — `player_state.gd`（enter/exit/process/physics_process のインターフェース）
- [ ] **P2-07** Idle ステート — `idle_state.gd`（待機アニメ再生、入力に応じた遷移判定: Walk/Attack/Jump/Dash）
- [ ] **P2-08** Walk ステート — `walk_state.gd`（8方向移動、X軸速度 + Y軸速度×0.6、向き切替、入力なしで Idle へ）
- [ ] **P2-09** 奥行きソート処理（Y座標による `z_index` 自動更新、または親ノードの YSort 実装）
- [ ] **P2-10** Attack ステート（コンボ1段目）— `attack_state.gd`（攻撃アニメ再生、Hitbox 有効化/無効化、コンボ受付ウィンドウ）
- [ ] **P2-11** コンボ2〜3段目の実装（受付時間内の再入力で次段へ遷移、最終段 or タイムアウトで Idle）
- [ ] **P2-12** Jump ステート — `jump_state.gd`（altitude 変数による擬似高さ、放物線軌道、Sprite.offset.y 反映、着地判定）
- [ ] **P2-13** JumpAttack ステート — `jump_attack_state.gd`（空中攻撃、着地時に硬直）
- [ ] **P2-14** プレースホルダースプライト作成（32×48px、Idle/Walk/Attack1-3/Jump/Hit/Down アニメーション、3色分）
- [ ] **P2-15** テストステージにプレイヤーを配置して動作確認（移動・コンボ・ジャンプの統合テスト）
